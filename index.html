The Eduba frame

We think about agent products in three layers. Roughly 60% of what looks like "agent work" is traditional code and database work: storing state, retrying idempotent API calls, managing a queue. Roughly 30% is rule-based logic: which tool fires when, which field maps to which CRM property, what the retry policy is when LinkedIn returns a 429. The last 10% is where an LLM actually earns its keep: writing the message, interpreting ambiguous intent, deciding which prospect to pursue next.

60%Traditional code and database work
30%Rule-based logic and routing
10%Genuine LLM work

Most teams put an LLM at all three layers and wonder why the agent hallucinates itself into a bad outbound message. Applied to Sliq's stack specifically, the question is whether your context layer separates those three concerns cleanly, or whether the model is doing work that a Postgres query and a switch statement would do faster, cheaper, and with fewer surprises.

The relevant case study

One recent engagement was with Feeld, a product company whose CTO brought us in for a scoped sprint. The deliverable was an Organizational Context Architecture and a Strategic Operations Framework. The useful part of that engagement, for a team your size, was the artifact that came out of it: a folder structure that described how the team's AI systems, humans, and code were supposed to hand off to each other. Small scope, concrete output, survived the hand-off when the team grew.

Feeld, product company, CTO buyer

Scoped sprint. Workshop, advisory calls, Organizational Context Architecture, Strategic Operations Framework. Shape maps cleanly to a Sliq-sized engineering team operationalizing agents across third-party APIs.

The relevant paper

Interpretable Context Methodology. Submitted to ACM TiiS. Describes agent context as a layered filesystem with measurable interpretability and reproducibility gains. The repo is public, MIT-licensed, and sits at github.com/RinDig/Interpretable-Context-Methodology-ICM-. It is exactly the abstraction a Sliq-style agent product needs once it crosses a couple hundred active users.
